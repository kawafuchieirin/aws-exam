# 基盤モデルと生成AIの要点

> AIP-C01 は「用語の正確な理解」と「ユースケースに適した手法の選択」が中心。

## 押さえる用語

- **基盤モデル (Foundation Model)**：大量データで事前学習され、多様なタスクに転用できる大規模モデル。
- **トークン**：モデルが処理するテキストの最小単位。料金・コンテキスト長はトークンで数える。
- **埋め込み (Embedding)**：テキスト等を意味を保ったベクトルに変換したもの。検索・分類に使う。
- **温度 (Temperature)**：高いほど出力が多様/創造的、低いほど決定的。

## カスタマイズ手法の選択

| 要件 | 手法 |
|---|---|
| 最新・社内固有の知識を回答に反映 | RAG（検索拡張生成） |
| 出力の口調・形式を例で誘導 | プロンプトエンジニアリング（Few-shot） |
| ドメイン特化の振る舞いを内在化 | ファインチューニング |
| 学習コストを抑えつつ微調整 | PEFT / LoRA |

> 判断の基本：**まずプロンプト → 足りなければRAG → それでも不足ならファインチューニング**。コストと運用負荷はこの順に上がる。

## AWS の主なサービス

- **Amazon Bedrock**：基盤モデルをAPIで利用。RAG（ナレッジベース）・エージェント・ガードレールを統合。
- **Amazon SageMaker**：独自モデルの構築・学習・デプロイ。JumpStartで基盤モデルも利用可。
- **Amazon Q**：業務向けの生成AIアシスタント。

## 責任あるAI（頻出）

- **公平性・バイアス**、**説明可能性 (Explainability)**、**透明性**、**プライバシー**。
- Bedrock の **ガードレール**で有害コンテンツやPII、禁止トピックをフィルタできる。

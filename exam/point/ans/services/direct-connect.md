# AWS Direct Connect の要点

> 重要度: ◎ ／ 第1分野ハイブリッド接続の最重要。BGP トラフィック制御と冗長性設計（SLA）が即答必須。

## これは何
- オンプレと AWS を**専用光ファイバ回線で直結**する物理接続。インターネット非経由で**低レイテンシ・安定帯域**を提供。
- DX ロケーションで顧客ルータと AWS ルータをクロスコネクト。「専用線・安定帯域・インターネット非経由」要件の正解。

## 試験頻出ポイント
- **VIF 三種**: **Private VIF**＝VGW/DXGW 経由で VPC（プライベートIP、広告100）／**Public VIF**＝S3・DynamoDB 等パブリックサービスへ**全リージョン**（広告1000、全広告 NO_EXPORT、推移不可）／**Transit VIF**＝DXGW→**TGW** でマルチVPC・マルチアカウント・マルチリージョン集約。
- **戻り（AWS→オンプレ・インバウンド）= LOCAL_PREF コミュニティで制御**。`7224:7300`=High（アクティブ）、`7224:7200`=Medium（既定・ECMP負荷分散）、`7224:7100`=Low（パッシブ）。高いほど優先。
- **行き（オンプレ→AWS・アウトバウンド）= オンプレ側で制御**。**more-specific（長いプレフィックス）広告が最優先**、**AS_PATH プリペンド**で非優先化。
- **BGP 経路選択順**: 最長プレフィックス → LOCAL_PREF（高優先） → AS_PATH（短優先） → MED（小優先） → ECMP。プレフィックス長と LOCAL_PREF が同じときに初めて AS_PATH が効く。
- **Active/Active**=全VIFに同タグ（例 `7224:7200`）で ECMP（ASN一致不要）。**Active/Passive**=アクティブ `7224:7300`＋パッシブ `7224:7100`。
- **DXGW** はグローバルリソース。1 DX から複数リージョン・複数アカウントへ到達（中国除く）。**VGW 同士は DXGW 経由で相互通信不可**（ハブにならない）。DXGW 自体は無料。
- **MACsec**＝IEEE 802.1AE の **L2 回線暗号化**。**Dedicated 10G/100G** のみ対応、スループット低下小。「DX でも暗号化必須」要件の解。
- **SiteLink**＝DX ロケーション間（拠点間）を AWS バックボーン経由で**リージョン非経由・最短接続**（AWS をグローバル WAN 化）。
- **冗長性 SLA**: 最大耐障害性=**99.99%**（2ロケーション以上・別デバイス独立接続）／高耐障害性=**99.9%**（2ロケーション・別デバイス2接続）／単一ロケーションは SLA なし。
- **VPN バックアップ**: TGW/VGW で DX 経路を優先（伝播）し、DX 断時に [Site-to-Site VPN](site-to-site-vpn.md) へ自動切替。

## 比較・選択の判断
| 要件 | 解答 |
|---|---|
| VPC へプライベートIP接続 | Private VIF（VGW/DXGW） |
| S3/DynamoDB 等へ全リージョン・非経由 | Public VIF |
| 多数VPC・マルチアカウント/リージョン集約 | Transit VIF → DXGW → TGW |
| 複数リージョン・アカウントへ1接続で到達 | DXGW（グローバル） |
| DX 回線の暗号化・低オーバーヘッド | MACsec（Dedicated 10G/100G） |
| DX でプライベート＋暗号化（MACsec不可時） | Private IP VPN over DX（IPsec） |
| 拠点間をリージョン非経由で接続 | SiteLink |
| 戻りトラフィックを優先側へ寄せる | LOCAL_PREF コミュニティ（7224:73xx/72xx/71xx） |
| 行きトラフィックを優先側へ寄せる | more-specific 広告・AS_PATH プリペンド（オンプレ側） |
| 本番ミッションクリティカル冗長 | 最大耐障害性（2ロケーション・別デバイス、99.99%） |
| コスト重視のバックアップ | DX＋S2S VPN フェイルオーバ |

## よく問われる上限・注意点（ひっかけ）
- **広告経路上限を超えると BGP セッションが idle ダウン**: Private/Transit VIF は **IPv4・IPv6 各100**、Public VIF は **1,000**。経路集約で収める。
- **Hosted 接続は VIF 1つのみ**・MACsec 一般に非対応・帯域はパートナー分割（50Mbps〜10G）。Dedicated は VIF 最大51・1/10/100/400Gbps・MACsec対応。
- **LOCAL_PREF コミュニティ（72xx/73xx/71xx）は戻り制御専用**。行き（アウトバウンド）には効かない（オンプレ側で制御）。
- **Public VIF スコープコミュニティ（91xx/92xx/93xx）は広告範囲の制御**で LOCAL_PREF とは無関係。Public VIF は私有 ASN を 7224 に置換するため外部へのプリペンドは効かない。
- **単一ロケーション 2 接続は高耐障害性(99.9%)であって最大耐障害性(99.99%)ではない**。99.99% は 2 ロケーション以上が必須。
- **VGW 同士は DXGW 経由で通信不可**（DXGW はハブにならない／推移ルーティング不可）。VPC間は TGW を使う。
- **MACsec は Hosted 不可・1G不可**。10G/100G の Dedicated のみ。
- BGP デフォルト hold 90秒・keepalive 30秒。BFD は各VIFで自動有効（最小検出300ms×3）。Amazon側 Public VIF の ASN は 7224 固定。
- コストは①ポート時間課金＋②**アウトバウンドデータ転送(DTO)**。DXGW 自体は無料。

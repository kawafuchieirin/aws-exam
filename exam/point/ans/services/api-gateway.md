# Amazon API Gateway の要点

> 重要度: ○ ／ ネットワーク観点ではエンドポイントタイプ・プライベートAPI(PrivateLink)・カスタムドメインが論点。第1分野(設計)・第4分野(セキュリティ)で頻出。

## これは何
- API の作成・公開・保護・監視を行うフルマネージドサービス。
- REST / HTTP / WebSocket の各タイプを提供し、Lambda・HTTPバックエンド・各AWSサービスへのフロントドアになる。
- 試験では「APIをどこに公開するか」「VPC内バックエンドへの統合」「非公開化」が中心。

## 試験頻出ポイント
- **エンドポイントタイプ = エッジ最適化 / リージョナル / プライベート**。**プライベート・エッジ最適化は REST API のみ**、HTTP/WebSocket は**リージョナルのみ**。
- **エッジ最適化は内部で CloudFront(AWS管理)を経由**し最寄りPOPへ。REST の既定タイプ。
- **プライベートAPI** はインターネット非公開。**`execute-api` Interface VPCエンドポイント(PrivateLink)経由のみ**で呼べる。
- プライベートAPIのアクセス制御は二段: **APIのリソースポリシー(`aws:SourceVpc`/`aws:SourceVpce`)が必須** + VPCエンドポイントポリシー。
- **プライベートDNS有効化**で `Host`/`x-apigw-api-id` ヘッダなしで呼べる。ただし有効化中はパブリックAPIのデフォルトエンドポイントへVPCからアクセス不可になる。
- **使用プラン + APIキー(クライアント単位レート制限・クォータ・課金)は REST API のみ**。
- **HTTP API** は REST より最大71%安・最大60%低レイテンシ。Lambda/HTTPプロキシ向き。管理機能は持たない。
- **VPCリンク**でVPC内プライベートバックエンドへ統合: **REST→NLB**、**HTTP→ALB/NLB/Cloud Map**。
- カスタムドメイン: **エッジ最適化は us-east-1 のACM証明書**、**リージョナルはそのリージョンのACM証明書**。
- マルチリージョン: リージョナルAPI×複数 + **Route 53 レイテンシ/フェイルオーバールーティング**。固定IPが要れば [Global Accelerator](global-accelerator.md) 併用。
- スロットル超過は **`429 Too Many Requests`**。適用順は **使用プラン → ステージ/メソッド → アカウント単位 → リージョン全体**。

## エンドポイントタイプの選択
| 要件 | タイプ |
|---|---|
| 世界中に地理的分散したクライアント | エッジ最適化(REST既定) |
| 同一リージョン/EC2 からの呼び出し、前段に自前CloudFront、Route 53レイテンシ併用 | リージョナル |
| VPC内/オンプレ(DX/VPN経由)からのみ呼ぶ・インターネット非公開 | プライベート(RESTのみ) |
| HTTP API / WebSocket API を使う | リージョナル一択 |

## よく問われる上限・注意点（ひっかけ）
- **HTTP/WebSocket はプライベート/エッジ最適化に非対応**(リージョナルのみ)。「HTTP APIをエッジ最適化に」は誤り。
- **プライベートAPIは TLS 1.2 のみ**。HTTP/2 は HTTP/1.1 に強制、IPタイプはデュアルスタックのみ(IPv4のみ指定不可)。
- プライベートAPIで「SGだけ」「WAFだけ」では制御不足。**リソースポリシー必須**が正答。
- **REST API の VPCリンクは NLB のみ**(ALB不可)。HTTP API は ALB/NLB/Cloud Map。混同に注意。
- エッジ最適化のカスタムドメインACMは必ず **us-east-1**(API のデプロイリージョンではない)。
- アカウント単位スロットル既定: **10,000 RPS + バースト 5,000**(一部新興リージョンは 2,500/1,250)。引き上げ申請可、バーストは顧客が直接変更不可。
- 統合タイムアウト: REST 最大 **29秒**、HTTP 最大 **30秒**。
- スロットル/クォータは**ベストエフォート**(厳密保証ではない)。
- プライベートAPIは Interface エンドポイントの時間+データ課金が別途発生。1つのVPCエンドポイントで複数プライベートAPIを共有できる(VPCe数削減)。

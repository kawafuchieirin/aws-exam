# VPC 設計の要点

> Amazon VPC の出題は「サブネット/ルーティング設計」と「VPC間・オンプレ接続の選択」が中心。

## サブネット設計

- **AZ をまたいで冗長化**：1サブネット = 1AZ。可用性のため最低2AZに分散。
- **パブリック/プライベートの違いはルートテーブル**：`0.0.0.0/0 → IGW` ならパブリック、`→ NAT` ならプライベート。
- **CIDR は重複させない**：将来のピアリング/TGW接続を見越して計画的に割り当てる。

## ルーティングの判断

| 要件 | 解答 |
|---|---|
| プライベートサブネットから外部へ送信のみ許可 | NAT Gateway |
| VPC内からS3/DynamoDBへプライベート接続 | Gateway VPC エンドポイント |
| VPC内から他AWSサービスへプライベート接続 | Interface VPC エンドポイント (PrivateLink) |
| オンプレと相互接続 | Direct Connect / Site-to-Site VPN |

## VPC間・ハイブリッド接続の選択

- **少数VPC・低コスト** → VPC ピアリング（推移的ルーティング不可に注意）。
- **多数VPC＋オンプレを集中管理** → Transit Gateway（ハブ＆スポーク）。
- **CIDR が重複したまま特定サービスだけ公開** → PrivateLink。

## よく問われる上限・制約

- VPC ピアリングは**推移的でない**（A–B, B–C があっても A–C は通らない）。
- セキュリティグループは**ステートフル**、ネットワークACLは**ステートレス**。
- 1つの NAT Gateway は1AZ内。AZ障害に備えてAZごとに配置する。

# Route 53 の要点

> 出題は「ルーティングポリシーの使い分け」と「ハイブリッドDNS（Resolver）」に集中する。

## ルーティングポリシー 即答表

| 要件 | ポリシー |
|---|---|
| 単純に1つのリソースへ | シンプル |
| 障害時に別リソースへ切替 | フェイルオーバー（ヘルスチェック必須） |
| 最も近いリージョンへ | レイテンシベース |
| 地理的な場所で振り分け | 地理的近接 / 位置情報 |
| 重み付けで分散（カナリア/AB） | 加重 |
| 複数の正常なレコードを返す | 複数値回答 |

## ハイブリッドDNS（Route 53 Resolver）

- **インバウンドエンドポイント**：オンプレ → AWS の名前解決（オンプレから VPC 内ホスト名を引く）。
- **アウトバウンドエンドポイント**：AWS → オンプレ の名前解決（条件付き転送ルールで委譲）。
- マルチアカウントでは **RAM** で Resolver ルールを共有する。

## プライベートホストゾーン（PHZ）

- VPC 内だけで有効な内部DNS。複数VPCに関連付け可能。
- VPC設定 `enableDnsHostnames` / `enableDnsSupport` が有効でないと解決されない。
